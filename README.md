# boristane-workflow

Boris Tane 工作流 + 最小三变体集成 (A/B/C) + 评审融合，用于复杂任务（>4 个子步骤）。提及 "boris" 时自动触发。

## 灵感来源

boristane-workflow 源于对复杂软件工程任务的系统化处理需求：
1. 任务复杂度评估困难
2. 方案设计单一
3. 评审缺乏量化标准
4. 实现与测试脱节

## 技术特点

### 1. 智能任务分类
- 简单任务 (≤4 子任务): 直接执行
- 复杂任务 (>4 子任务): 启用完整多代理团队协作

### 2. 动态 Agent 团队
- Leader: 统筹规划
- Reviewer: 量化评审
- Executor: 具体执行

### 3. 三变体方案
- Variant A: 极简版
- Variant B: 鲁棒版
- Variant C: 性能版

### 4. TDD 强制执行
先写测试，测试通过才算完成

### 5. 量化评审
5个维度各20分：代码质量、可维护性、性能、安全性、可扩展性
